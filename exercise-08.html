Calculating this by hand, from the formulas of slope = cov(x,y)/var(x) and Intercept (beta0) = y’ - beta1(x’),

I am using the raw formulas of cov(x,y) and var(x) and direct fucntions as well to calculate this, Both ways give the similar results.

x <- d1$Group_size # Wecan decide what is going to be the predictir and response by assigning to x and y
y <- d1$ECV

variance_x = sum((x - mean(x))^2)
covariance_x_y = sum((x - mean(x)) * (y - mean(y)))

beta1 <- covariance_x_y / variance_x

beta0 <- mean(y) - beta1 * mean(x)

beta1 <- cov(x, y) / var(x)
beta0 <- mean(y) - beta1 * mean(x)

calculating the standard error for the slope coefficient, the 95% CI, and the p value associated with this coefficient by hand. I can use the discussed formulas during the class.

yhat <- beta0 + beta1 * x
residual <- y - yhat
n <- length(x)

Calculating standard error,

SE_beta1 <- sqrt(sum(residual^2) / ((n - 2) * sum((x - mean(x))^2)))

t_beta1 <- beta1 / SE_beta1

p_beta1 <- 2 * (1 - pt(abs(t_beta1), df = n - 2))

t_crit <- qt(0.975, df = n - 2)

Step 07

Now with 1000 permutations for a null sampling distribution of the slope. Under the null hypothesis, we can say that there is no relationship between Group_size and ECV. So what I need is to break the association between them by permutting one variable relative to the other

To do that, first I need observed slope

observed_slope <- coef(model1)[2]
observed_slope
Group_size 
  2.463071

And, now I can generate permutation null distribution

n_perm <- 1000

perm_slopes <- replicate(n_perm, {
  y_perm <- sample(d1$ECV)
  coef(lm(y_perm ~ d1$Group_size))[2]
})

Looking at what we got,

hist(perm_slopes)

now getting the permutation P value from the permutation distribution, we can see that the p value is zero

permutation_p_val <- mean(abs(perm_slopes) >= abs(observed_slope))
permutation_p_val
[1] 0

The permutation approach tests the null hypothesis by breaking the association between variables. We randomly shuffle ECV values here so that any real association is removed by that.

By getting a zero for the permutation p value I can say that the observed slope is away from the distribution.

Step 08

Using bootstrapping to generate a 95% CI for my estimate of the slope coefficient using

1) the quantile method

I am using the exact code we tried during the Tuesday’s class. We can use bootstrapping because it’s resampling can estimate the variability of the slope. I do not base on any theoretical assumptions here.

n_boot <- 1000

boot_slopes <- replicate(n_boot, {
  boot_rows <- sample(1:nrow(d1), size = nrow(d1), replace = TRUE)
  d_boot <- d1[boot_rows, ]
  coef(lm(ECV ~ Group_size, data = d_boot))[2]
})

Now looking at it,

hist(boot_slopes)

Let’s now calculate bootstrap CI using quantile method (also the same code in the Tuesday’s class)

bootstrap_ci_quantile <- quantile(boot_slopes, c(0.025, 0.975))
bootstrap_ci_quantile
    2.5%    97.5%

2) Using theory based method

SE_boot <- sd(boot_slopes)

bootstrap_ci <- c(
  observed_slope - qt(0.975, df = n - 2) * SE_boot,
  observed_slope + qt(0.975, df = n - 2) * SE_boot
)

bootstrap_ci

Clearly zero is not inside any of our calculated CIs. This means our slope cannot be zero, it should be different than zero. This actually provides evidence for the positive relationship between social group size and brain size.
